INRIX Client Library for iOS
Route Notifications

The INRIX Client Library exposes several NSNotification messages and delegate subscriptions that can be used to learn about changes to route status.

Incident Alerts

The IXCLDriveTimeAlertsIncidentsOnRouteNotification message is sent when there are new incidentes on the current active route.
The notification object is an NSDictionary with the following keys:

  • IXCLDriveTimeAlertIncidentOnRouteCurrentRouteKey - An IXCLRoute object for the current route
  • ISCLDriveTimeAlertIncidentOnRouteIncidentsArrayKey - An NSArray of IXCLIncident objects

Faster Route Alerts

The IXCLRouteUpdateFasterRouteAvailableNotification message is sent when the current route has a faster alternate available.
The notification object is an NSDictionary with the following keys:

  • IXCLRouteUpdateCurrentRouteKey - An IXCLRoute object for the current route
  • IXCLRouteUpdateFasterRouteKey - An IXCLRoute object for the faster alternate

Off-Route and Route-End Alerts

The IXCLRouteTrackingStatusChangedNotification message is sent when a route status changes.
The notification object is an NSDictionary with the following keys:

  • IXCLRouteTrackingStatusRouteKey - An IXCLRoute object for the current route
  • IXCLRouteTrackingStatusLocationKey - A CLLocation object with the relevant location
  • IXCLRouteTrackingStatusStatusKey - An IXCLRouteTrackingStatus enumeration indicating the new status. Possible values are:
    – IXCLRouteTrackingStatusOffRoute - The driver has driven off the selected route.
    – IXCLRouteTrackingStatusAtDestination - The drivr has reached the end of the selected route.

Updated ETA and Conditions

The IXCLRouteUpdateController exposes an addSubcriber method that accepts a delegate that implements the <IXCLRouteUpdateSubscriber> protocol. The subscriber will receive notifications when the route changes, including udpates to ETA, traffic conditions, speed buckets, etc.
