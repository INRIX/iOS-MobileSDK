◆ preferredPrimaryRouteIndex

- (NSInteger) preferredPrimaryRouteIndex
readwriteatomicassign

The preferred index of the primary route.

Discussion
This index specifies a preference for the primary route. Valid values are between 0 and one less than the count of routes that were most recently published, IXCLRouteUpdateControllerPrimaryIndexDefault indicating that the fastest route is the preferred primary route, or IXCLRouteUpdateControllerPrimaryIndexNone indicating that there is no primary route. Examples of how this could be used include drawing the primary route differently or using only the primary route for processing.

When routes are requested and then published to the subscribers, the actual value passed to the subscribers could be different based on the number of routes received from the server. For example, if the value set is 1 and only one route is received from the server, a value of 0 will be passed to subscribers.

When setting, subscribers will be updated with the new value.

Defaults to IXCLRouteUpdateControllerPrimaryIndexDefault.

Exceptions
IXCLExceptionDomainThe specified value is not in the range of valid values.
